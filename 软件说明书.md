# 标书智写软件说明书

本文面向 Windows 普通用户，说明如何初始化、配置和使用 `bid-writer.exe`。

## 一、软件包内容

解压后的目录建议保持如下结构：

```text
bid-writer/
  bid-writer.exe
  config.yaml
  config.example.yaml
  .env.example
  README.md
  软件说明书.md
  启动命令.txt
  outline.md
  roles/
    system_gate_rules.md
    通用投标角色.md
    example_role.md
  项目要求/
    项目采购需求.md
    评分标准.md
  output/
  log/
  caches/
```

其中：

- `bid-writer.exe`：软件主程序。
- `config.yaml`：默认运行配置，首次启动会优先读取它。
- `config.example.yaml`：配置模板，配置损坏时可复制恢复。
- `.env.example`：模型连接参数模板。
- `roles/system_gate_rules.md`：系统门禁规则文件，生成正文时必须存在。
- `roles/通用投标角色.md`：默认通用投标身份文件。
- `outline.md`：默认投标大纲文件。
- `项目要求/项目采购需求.md`：默认采购需求文件。
- `项目要求/评分标准.md`：默认评分标准文件。
- `output/`：生成结果默认保存目录。
- `log/`：生成过程日志和 trace 默认保存目录。
- `caches/`：自动提炼、背景证据等缓存目录。

## 二、首次初始化

1. 将压缩包完整解压到一个固定目录，例如：

```text
D:\bid-writer
```

2. 确认 `bid-writer.exe` 与 `config.yaml` 在同一层目录。

3. 在同一层目录中新建 `.env.local` 文件。

可以先复制 `.env.example`，再重命名为 `.env.local`。`.env.local` 用来保存真实模型地址和 API Key，不建议发给他人。

## 三、设置模型连接

用记事本打开 `.env.local`，至少填写以下内容：

```env
BID_WRITER_API_BASE_URL=https://api.openai.com/v1
BID_WRITER_MODEL=gpt-5.4
BID_WRITER_API_KEY=你的真实API_KEY
BID_WRITER_TEMPERATURE=0.7
BID_WRITER_MAX_TOKENS=10000
BID_WRITER_TIMEOUT_SECONDS=120
BID_WRITER_MAX_RETRIES=3
```

如果使用兼容 OpenAI Chat Completions 的第三方模型服务，把 `BID_WRITER_API_BASE_URL`、`BID_WRITER_MODEL` 和 `BID_WRITER_API_KEY` 改成服务商提供的值。

可选的辅助模型、向量检索和 embedding 参数也可以写在 `.env.local` 中，字段名可参考 `.env.example`。

## 四、准备项目文件

默认配置读取以下三个文件：

- `outline.md`
- `项目要求/项目采购需求.md`
- `项目要求/评分标准.md`

使用新项目时，可以直接替换这三个文件的内容：

1. 将投标文件目录或写作大纲粘贴到 `outline.md`。
2. 将招标文件中的采购需求粘贴到 `项目要求/项目采购需求.md`。
3. 将评分办法、评分细则或技术评分表粘贴到 `项目要求/评分标准.md`。

大纲建议使用 Markdown 标题层级：

```markdown
# 项目总标题
## 一级章
### 二级节
#### 具体写作单元
```

系统会优先选择最深层叶子标题作为具体写作单元。

## 五、设置投标主体和角色

打开 `config.yaml`，重点检查以下字段：

```yaml
project:
  root_dir: "."
  bidder_name: "示例投标主体名称"
  inputs:
    outline_file: "./outline.md"
    bid_requirements_file: "./项目要求/项目采购需求.md"
    scoring_criteria_file: "./项目要求/评分标准.md"
  output_dir: "./output"

writing:
  role_file: "./roles/通用投标角色.md"
```

需要修改的常见项：

- `project.bidder_name`：改成实际投标主体名称。
- `project.inputs.outline_file`：大纲文件路径。
- `project.inputs.bid_requirements_file`：采购需求文件路径。
- `project.inputs.scoring_criteria_file`：评分标准文件路径。
- `project.output_dir`：输出目录。
- `writing.role_file`：角色身份文件，默认使用 `./roles/通用投标角色.md`。

如果不同项目需要不同身份，可以复制 `roles/通用投标角色.md` 为新的角色文件，然后在 `config.yaml` 中修改 `writing.role_file`。

`roles/system_gate_rules.md` 是固定门禁规则文件，不建议删除或改名。

## 六、启动软件

双击运行：

```text
bid-writer.exe
```

启动时软件会按以下顺序寻找配置：

1. 上次成功使用的配置。
2. 当前目录下的 `config.yaml`。
3. 当前目录下其它 `config*.yaml` 或 `config*.yml`。

如果没有自动加载到正确配置，可在软件界面中选择或浏览 YAML 配置文件。

## 七、基本使用流程

1. 启动软件后，确认顶部显示的配置文件正确。
2. 点击或使用菜单重载大纲，确认左侧章节树已经显示。
3. 在左侧选择一个具体章节。
4. 设置目标篇幅、表格数量、流程图数量等生成参数。
5. 点击生成当前章节，等待模型返回。
6. 检查生成结果，必要时补充附加要求后重新生成。
7. 批量生成时，建议先用单个章节测试模型连接和写作风格。
8. 生成文件会保存到 `output/` 目录。

## 八、输出文件

默认输出目录是：

```text
output/
```

单章节文件名会根据章节标题生成，并自动处理 Windows 文件名中的非法字符。

如果启用了 trace，生成过程中的提示词、上下文和输出摘要会保存到：

```text
log/generation_traces/
```

trace 用于排查问题，可能包含项目文本，不建议直接对外发送。

## 九、常见问题

### 1. 启动后提示找不到配置文件

确认 `bid-writer.exe` 同级目录存在 `config.yaml`。如果没有，可以复制 `config.example.yaml` 并重命名为 `config.yaml`。

### 2. 生成时报 system gate rules 文件不存在

确认以下文件存在：

```text
roles/system_gate_rules.md
```

该文件必须位于配置文件同级目录下的 `roles/` 文件夹内。

### 3. 生成时报角色文件不存在

检查 `config.yaml` 中的：

```yaml
writing:
  role_file: "./roles/通用投标角色.md"
```

确认对应文件存在，或把路径改成实际角色文件。

### 4. 模型连接失败或 API Key 无效

检查 `.env.local` 是否存在，并确认：

- `BID_WRITER_API_BASE_URL` 是否正确。
- `BID_WRITER_MODEL` 是否正确。
- `BID_WRITER_API_KEY` 是否填写真实有效的密钥。
- 当前网络是否能访问模型服务。

### 5. 大纲、采购需求或评分标准没有读到

检查 `config.yaml` 中的路径是否正确。默认相对 `project.root_dir` 解析。

如果 `project.root_dir` 是 `"."`，表示相对当前软件目录。

### 6. 生成内容不够贴合评分标准

建议先检查 `项目要求/评分标准.md` 是否包含完整评分细则。也可以在生成前补充附加要求，例如“重点响应评分标准中的服务方案完整性、人员配置和质量控制要求”。

### 7. 不要把哪些文件发给别人

不要发送以下文件：

- `.env.local`
- `.env`
- 包含真实 API Key 的任何截图或配置文件
- `log/generation_traces/` 中含有敏感项目内容的日志
- 已生成但不适合外发的投标正文

## 十、推荐使用习惯

- 每个投标项目单独复制一份软件目录，避免不同项目文件互相覆盖。
- 先生成 1 个章节检查风格，再批量生成。
- 修改大纲后重新加载大纲。
- 修改 `.env.local` 后重启软件。
- 保留 `config.example.yaml`，方便配置异常时恢复。
