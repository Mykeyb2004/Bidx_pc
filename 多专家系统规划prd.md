## 一、项目概述

### 1.1 产品定义

**投标方案智能生成系统**是一个基于 CrewAI 的多专家协作系统，专门用于：
- 深度解析招标项目的评分标准
- 识别高权重评分项并制定得分策略
- 分析项目需求与招标方真实诉求
- 生成高质量、高响应、高得分的投标方案大纲

### 1.2 核心价值

| 价值点 | 说明 |
|-------|------|
| 精准识别 | 从复杂评分标准中提取核心要点 |
| 策略导向 | 针对评标专家关注点进行策略性展开 |
| 专业框架 | 生成符合招标方预期的专业方案结构 |
| 效率提升 | 将方案策划时间从数天缩短至数小时 |

### 1.3 目标用户

- **企业投标负责人**：需要快速理解招标文件并制定响应策略
- **售前解决方案架构师**：需要设计符合评分标准的技术方案
- **商务人员**：需要制定有竞争力的报价策略

## 二、专家团队设计（Agents）

| 编号 | Agent 名称 | 角色定位 | 核心能力 | 依赖 |
|-----|-----------|---------|---------|------|
| A1 | 评标分析师 | 招标文件解析专家 | 评分标准提取、权重分析 | 无 |
| A2 | 需求解读专家 | 需求理解专家 | 隐性需求挖掘、需求解构 | A1 |
| A3 | 策略规划师 | 得分策略专家 | 得分最大化、风险防控 | A1, A2 |
| A4 | 技术架构师 | 技术方案设计师 | 架构设计、技术选型 | A3 |

| A6 | 方案整合师 | 方案整合专家 | 结构化输出、大纲生成 | A1-A5 |

### 2.2 详细 Agent 定义

```yaml

  role: "招标项目评标分析师"

    曾参与过200+大型项目的评标工作，精通各类评标办法（综合评分法、经评审最低价法等）。
    你擅长从复杂的评分标准表格中提取核心要点，识别评标专家的关注重点。
  traits:
    - 精通《政府采购法》《招标投标法》及相关法规
    - 熟悉各行业评分标准特点（IT、建筑、服务等）

    verbose: true
```

- PDF 解析工具

### 3.2 阶段1：文件解析

#### T1: 评分标准分析 (analyze_bidding_documents)

**输入**：
- 招标文件 PDF/Word 文件
- 评分标准附件

**处理逻辑**：
1. 提取评分标准表格
2. 计算各评分项权重占比
3. 识别高权重评分项（≥10分）
4. 分析评分细则中的隐性要求

**输出**：`analysis/01_bid_evaluation_analysis.md`

```markdown
## 招标文件评分标准分析报告

### 一、评分标准总览
| 序号 | 评分项目 | 分值 | 权重 | 评分方式 | 关键要点 |
|------|---------|------|------|---------|---------|
| 1    | 技术方案 | 40   | 40%  | 评分细则 | 重点展开 |
| 2    | 项目团队 | 20   | 20%  | 评审打分 | 资质证明 |
| ...  | ...     | ...  | ...  | ...     | ...     |

### 二、高权重评分项（TOP 5）
1. **技术方案设计**（40分）- 核心得分项
2. **项目团队配置**（20分）
3. **企业资质业绩**（15分）
4. **商务报价**（15分）
5. **售后服务承诺**（10分）

### 三、评分细则深度分析
[针对每个高权重项的详细分析]

### 四、风险提示与应对策略
[可能的扣分点和规避方案]
```

#### T2: 需求解读 (interpret_requirements)

**输入**：
- 招标文件
- T1 输出

**处理逻辑**：
1. 提取功能需求清单
2. 识别关键技术指标
3. 挖掘隐性需求
4. 分析需求与评分的关联

**输出**：`analysis/02_requirements_interpretation.md`

#### T3: 得分策略制定 (develop_scoring_strategy)

**输入**：
- T1 输出
- T2 输出

**处理逻辑**：
1. 设定得分目标
2. 制定高权重项展开策略
3. 识别风险点并制定防控措施
4. 规划资源分配

**输出**：`analysis/03_scoring_strategy.md`

### 3.3 阶段2：方案设计

#### T4: 技术框架设计 (design_technical_solution)

**输入**：T3 输出

**输出**：`design/04_technical_framework.md`

#### T5: 商务框架设计 (design_business_proposal)

**输入**：T3 输出

**输出**：`design/05_business_framework.md`

### 3.4 阶段3：整合输出

#### T6: 方案大纲整合 (integrate_proposal_outline)

**输入**：T1-T5 所有输出

**输出**：`output/00_proposal_outline.md`

**核心内容**：
- 完整投标文件目录结构
- 各章节与评分标准对应表
- 高权重项展开指南
- 评标专家关注点提示

## 五、技术实现

### 5.1 CrewAI 配置

```python
# crews.yaml
bidding_proposal_crew:
  agents:
    - bid_analyst
    - requirements_interpreter
    - strategy_planner
    - technical_architect
    - business_expert
    - proposal_integrator
  tasks:
    - analyze_bidding_documents
    - interpret_requirements
    - develop_scoring_strategy
    - design_technical_solution
    - design_business_proposal
    - integrate_proposal_outline
  process: sequential
  verbose: true
  memory: true
  planning: true
  manager_llm: "gpt-4"
```

### 5.2 工具定义

| 工具名称 | 功能 | 返回类型 |
|---------|------|---------|
| parse_bidding_document | 解析招标文件 | dict |

| design_outline | 生成方案大纲 | str |
| extract_tables | 提取评分标准表格 | List[Table] |
