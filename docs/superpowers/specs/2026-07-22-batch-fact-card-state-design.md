# 批量事实卡片状态与统一应用设计

## 目标

修复批量生成参数弹窗把全局事实卡片显示为“整批默认勾选”所造成的误导，同时保留批量统一修改章节事实卡片引用关系的能力。直接开始批量扩写不得修改章节配置；只有用户进入独立统一配置窗口并确认后才写回配置。

## 现状问题

- 多章节弹窗把事实卡片模式硬编码为启用，并用单章节选择面板展示卡片。
- 全局卡片在单章节面板中默认勾选，因此看起来像会应用到全部所选章节。
- 实际批量生成会忽略面板临时选择，按各章节已保存的 `should_reference/selections` 执行。
- “保存事实卡片引用关系”会把一个面板状态覆盖到全部章节，但界面没有把这项高影响操作与普通生成参数区分开。

## 选定方案

### 批量生成参数弹窗

- 不再显示“批量生成启用事实卡片模式”复选框。
- 不再创建 `FactCardSelectionPanel`。
- 显示只读聚合摘要：所选章节总数、启用事实卡片模式的章节数，以及每张 active 卡片的有效引用章节数。
- 同时存在引用与不引用时显示“混合状态”。
- 新增“本次批量忽略全部事实卡片”复选框，默认不选；它只把本次调用的 `fact_card_mode` 置为 `False`，不写配置。
- 新增“统一应用到所选章节…”按钮，打开独立统一配置窗口。

### 独立统一配置窗口

- 事实卡片模式提供三个选择：`保持各章节`、`全部启用`、`全部关闭`。
- 每张 active 卡片提供三个选择：`保持各章节`、`全部引用`、`全部排除`。
- 默认全部为“保持各章节”，因此打开窗口后直接确认不会改变任何配置。
- 确认前展示将修改的章节数、模式操作和卡片操作；没有任何变更时只提示无需应用。
- 用户取消确认或关闭窗口时不写配置。
- 应用成功后刷新批量弹窗中的聚合摘要。

### 批量补丁语义

- 模式补丁 `None` 表示保持，`True` 表示全部启用，`False` 表示全部关闭。
- 卡片补丁只包含用户明确选择“全部引用/全部排除”的卡片；未出现在补丁中的卡片保持原值。
- 全局卡片“全部引用”删除章节中的显式排除记录，“全部排除”写入 `selected: false`。
- 局部卡片“全部引用”写入 `selected: true`，“全部排除”删除章节中的选中记录。
- 关闭事实卡片模式时保留章节卡片选择，便于后续重新启用。
- 一次批量应用只加载一次 YAML、在内存中更新全部目标章节、最后保存一次，避免部分章节写入成功。

## 数据与兼容性

- 不新增或修改 YAML 字段。
- 继续使用 `chapter_defaults.<path>.should_reference` 和 `selections`。
- 旧版列表格式仍由现有读取逻辑兼容；批量保存后使用当前规范结构。
- 单章节弹窗、单章节自动保存、事实卡片解析和 prompt 注入逻辑保持不变。

## 错误处理

- 批量应用失败时显示错误对话框，保持统一配置窗口打开，不刷新摘要。
- 确认取消不调用存储层。
- inactive 或不存在的卡片补丁在存储层忽略，防止写入悬空引用。

## 测试与验收

- 纯聚合测试覆盖启用/关闭混合状态、全局排除、局部选中和无配置章节。
- 存储测试覆盖保留未修改项、全局引用/排除、局部引用/排除、模式开关以及单次保存。
- 对话框测试覆盖默认保持、确认取消、无变更不写入、成功刷新摘要。
- 生成参数测试覆盖批量弹窗不创建单章节选择面板、默认不忽略、本次忽略不写配置、单章节行为不变。
- 使用 `config_松滋助联体.yaml` 验收时，“人员配置”应显示 `16/46` 且为混合状态；直接开始扩写后配置文件内容不变。
