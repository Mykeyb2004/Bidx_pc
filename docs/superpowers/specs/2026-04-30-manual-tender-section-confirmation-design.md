# 招标文件资料人工确认设计

## 背景

当前“新建配置...”向导的“开始抽取”会把招标文件转换为 Markdown，再自动定位“项目采购需求”和“评分标准”。自动定位能提升效率，但招标文件标题、目录、表格和章节边界不稳定，算法可能把目录项、父章节或相邻内容误认为目标正文。

本设计把抽取链路改为“自动定位 + 人工确认”。算法只负责默认定位和默认选中，正式写入 `项目要求/项目采购需求.md` 与 `项目要求/评分标准.md` 必须由用户确认触发。

## 目标

- 点击“开始抽取”后必须打开人工确认窗口，不允许静默写入资料文件。
- 自动定位成功时，窗口默认跳转并选中算法识别出的目标内容。
- 自动定位失败时，窗口仍然打开，并允许用户从全文手动选择。
- 用户可以在渲染后的 Markdown 阅读视图中按块选择，也可以在 Markdown 源码区精确拖选。
- “项目采购需求”和“评分标准”分两步确认，分别点击按钮后立即写入对应文件。
- 覆盖已有非空文件时沿用现有确认覆盖和 `.bak` 备份策略。
- 保留现有导入报告、转换产物和配置路径填充流程。

## 非目标

- 不做 PDF、Word 原版页面预览和页面框选。
- 不引入浏览器或 WebView 作为正式应用依赖。
- 不在本次设计中重写章节定位算法，只扩展其结果供人工确认使用。
- 不对用户选择的内容做摘要、改写或结构化重排；保存用户确认的 Markdown 原文。

## 用户流程

用户在新建配置向导的“项目材料”步骤点击“开始抽取”。

系统先执行现有转换和自动定位：

1. 将 PDF、Word 或 Excel 招标文件转换为 Markdown 与 block 列表。
2. 使用现有章节抽取器生成“项目采购需求”和“评分标准”的候选结果。
3. 写入导入报告，但不写入正式资料文件。
4. 打开“招标文件资料确认”窗口。

确认窗口固定两步：

1. 第 1 步：项目采购需求
   - 若有自动定位结果，默认跳转到该区间并选中。
   - 若无自动定位结果，提示“未自动定位，请手动选择”，从全文开头显示，不默认选中。
   - 用户确认或调整选区后，点击“存入项目需求”。
   - 系统立即写入 `项目要求/项目采购需求.md`。

2. 第 2 步：评分标准
   - 流程与第 1 步一致。
   - 用户点击“存入评分标准”后，系统立即写入 `项目要求/评分标准.md`。

两步都完成后，窗口关闭，向导把资料路径填入：

- `./项目要求/项目采购需求.md`
- `./项目要求/评分标准.md`

如果用户取消窗口：

- 已点击“存入”的文件保留。
- 未确认的文件不写入。
- 向导停留在项目材料步骤，并提示本次确认未完成。

## 窗口结构

确认窗口采用三栏双视图布局。

左侧为确认控制栏：

- 当前步骤：`第 1/2 步 项目采购需求` 或 `第 2/2 步 评分标准`。
- 自动候选摘要：标题、置信度、警告信息。
- 候选切换：上一候选、下一候选。
- 选区辅助：扩大到上一段、扩大到下一段、收缩到当前章节。
- 操作按钮：取消、存入项目需求、存入评分标准。

中间为渲染后的 Markdown 阅读视图：

- 标题、段落、列表、表格尽量按阅读效果展示。
- 自动命中的内容以高亮背景显示。
- 用户可以点击章节、段落、列表项或表格块，将其加入或替换当前选区。
- 阅读视图的目标是“看得清楚”和“块级选择”，不是实现完整 Markdown 编辑器。

右侧为 Markdown 源码精确选择区：

- 展示转换后的 Markdown 全文。
- 自动命中时同步选中对应源码范围并滚动到附近。
- 用户可以像文本编辑器一样拖选任意连续 Markdown 文本。
- 最终保存内容以源码区当前选区为准。

## 选择规则

默认选区来源于自动抽取结果的 `start_block_id` 与 `end_block_id`。系统通过 block 顺序映射到完整 Markdown 文本中的字符范围。

当用户在渲染视图点选 block 时：

- 若没有选区，设置为该 block。
- 若已有选区且新 block 邻近当前选区，扩展为连续 block 区间。
- 若新 block 距离当前选区较远，提示是否替换当前选区。

当用户在源码区拖选时：

- 源码选区成为最终保存依据。
- 阅读视图根据选区覆盖到的 block 做近似高亮。

辅助按钮规则：

- 扩大到上一段：把选区开始移动到上一个 block 的起点。
- 扩大到下一段：把选区结束移动到下一个 block 的终点。
- 收缩到当前章节：若选区内包含标题 block，收缩到该标题及其同级标题之前的内容；否则保持当前选区并提示无法收缩。
- 上一章节 / 下一章节：优先按章节边界配置识别出的完整章节范围跳转，而不是按当前选区行数等距平移；只有全文没有可用章节边界时，才退回行数平移兜底。

保存前校验：

- 选区去除空白后不能为空。
- 若选区少于 20 个有效字符，提示内容过短，用户可继续保存或返回调整。
- 对评分标准，若没有“评分、评审、分值、满分、权重、得分”等词且没有 Markdown 表格，提示可能选错，用户可继续保存或返回调整。
- 对项目采购需求，若没有“服务、技术、要求、内容、范围、参数、成果、验收、采购”等词，提示可能选错，用户可继续保存或返回调整。

## 数据流

`TenderImportService` 需要从“转换抽取并直接写入”调整为可插入人工确认：

1. 转换招标文件，得到 `TenderConversionResult`。
2. 调用抽取器得到 `TenderSectionExtraction`。
3. 写入 `extraction_report.json`。
4. 调用人工确认回调，传入转换结果、抽取结果和目标文件信息。
5. 人工确认回调返回每个 section 的最终 Markdown 内容，或返回取消状态。
6. 服务层调用现有 `_write_target()` 写入目标文件。

建议新增模型：

```python
@dataclass(frozen=True)
class ManualTenderSectionSelection:
    section_key: str
    markdown: str
    start_block_id: str | None
    end_block_id: str | None
    manually_adjusted: bool

@dataclass(frozen=True)
class ManualTenderConfirmationResult:
    requirements: ManualTenderSectionSelection | None
    scoring: ManualTenderSectionSelection | None
    cancelled: bool = False
```

`TenderSectionExtraction` 继续表示算法结果。人工确认结果不覆盖算法结果，而是作为最终写入输入，便于报告和调试区分“算法判断”和“用户确认”。

导入报告可以扩展记录：

- 自动候选标题、范围、置信度和警告。
- 用户最终选区范围。
- 用户是否手动调整。

## 组件边界

### `bid_writer/tender_import_service.py`

- 保留转换、抽取、报告和写入职责。
- 新增 `confirm_sections` 回调，替代当前只在低置信度时使用的 `confirm_low_confidence`。
- 不依赖 Tk 细节，便于单元测试。
- 写入仍由服务层统一处理，继续复用覆盖确认和备份逻辑。

### `bid_writer/tender_import_dialog.py`

- 从低置信度 yes/no 弹窗升级为人工确认窗口。
- 负责 Tk 界面、双视图同步、选区调整和保存前提示。
- 对外提供类似 `confirm_tender_sections(parent, conversion, extraction, targets)` 的函数。

### `bid_writer/tender_section_extractor.py`

- 继续提供自动定位结果。
- 如需要支持“上一候选 / 下一候选”，可在现有 `candidates` 中保留同 section 的排序候选。
- 不在本次改动中大幅重写打分策略。

### `bid_writer/new_config_wizard.py`

- “开始抽取”调用新确认流程。
- 导入成功后继续设置 `requirements_path` 和 `scoring_path`。
- 用户取消人工确认时，状态提示改为“已取消确认，未完成资料写入”。

## 错误处理

- 转换失败：不打开确认窗口，按现有导入失败提示处理。
- 自动抽取不完整：仍打开确认窗口，未命中的 section 要求手动选择。
- 用户取消确认窗口：返回 cancelled，未写入的目标保持不变。
- 覆盖确认取消：当前 section 写入失败并提示，用户留在确认窗口或返回向导。
- 写入异常：展示错误，保留窗口状态，用户可重试或取消。

## 测试计划

使用 `uv run pytest`。

服务层测试：

- 自动抽取完整时，若未提供人工确认结果，不写入目标文件。
- 人工确认返回两段 Markdown 后，写入两个目标文件。
- 人工确认取消时，返回 cancelled，不写入未确认文件。
- 覆盖已有非空文件时继续生成 `.bak`。
- 抽取不完整但人工确认补齐内容时可以写入。

选择模型测试：

- 根据 `start_block_id` 和 `end_block_id` 映射默认源码选区。
- 扩大到上一段和下一段能正确移动 block 边界。
- 源码拖选后保存内容以源码选区为准。
- 空选区和过短选区会触发提示逻辑。

GUI 轻量测试：

- 点击“开始抽取”会调用人工确认流程，而不是低置信度 yes/no 弹窗。
- 确认完成后向导路径字段更新。
- 取消确认后向导停留在项目材料步骤并显示状态。

## 验收标准

- 每次点击“开始抽取”后，只要转换成功，就必须出现人工确认窗口。
- 不经过用户点击“存入项目需求”和“存入评分标准”，正式资料文件不得被写入。
- 默认定位正确时，用户可以直接连续点击两次存入完成导入。
- 默认定位错误时，用户可以通过渲染视图块选或源码区拖选修正内容。
- 算法完全未命中时，用户仍能手动选择并写入。
- 现有覆盖备份行为保持不变。
