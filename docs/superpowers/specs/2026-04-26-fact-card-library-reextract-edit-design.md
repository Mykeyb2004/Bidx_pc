# 事实卡片库编辑重提炼设计

## 背景

事实卡片库已经改为列表窗口，点击“编辑当前卡片”后打开单卡编辑弹窗。新的产品口径要求：编辑已有卡片时应复用“提炼章节事实卡片”窗口，并允许对有章节来源的卡片重新提炼。

## 目标

- 从事实卡片库编辑已有卡片时，使用与“提炼章节事实卡片”一致的工作台体验。
- 对 `source.type=chapter_extract` 且有 `source.chapter_path` 的卡片，允许用户修改提炼要求后重新提炼。
- 对手工卡片或缺少章节路径的卡片，仍可人工编辑，但不提供重新提炼能力。
- 保存时只更新当前卡片，不误删或重写同章节其他事实卡片。
- 重提炼保存后保留当前卡片 `id` 与来源章节路径，并更新 `source.extraction_instruction`。

## 设计

新增一个卡片库编辑工作台对话框，继承“提炼章节事实卡片”窗口的结构：顶部上下文说明、提炼要求区、单卡草稿编辑区、保存/取消按钮。该窗口接受当前卡片、初始草稿、可选提炼回调和来源说明。

对于章节提炼卡片：

- 初始提炼要求来自 `card.source.extraction_instruction`，为空时使用默认提炼要求。
- `extract_callback` 使用 `extract_fact_card_drafts_from_output_with_diagnostics()` 或旧版 `extract_fact_card_drafts_from_output()`，并通过 `card.source.chapter_path` 解析章节。
- 用户可直接编辑当前草稿保存，也可点击“重新提炼”后保存新草稿。

对于手工卡片：

- 使用同一工作台版式展示草稿编辑区。
- 提炼要求区显示只读提示，并禁用“重新提炼”。
- 保存仍通过单卡草稿更新完整卡片库。

保存时不调用 `replace_extracted_fact_cards()`，因为该接口面向“整章提炼结果替换”，会影响同章节其他提炼卡片。卡片库编辑保存走单卡库更新流程，必要时更新当前卡片的提炼要求。

## 测试

- 卡片库编辑章节提炼卡片时，应打开重提炼工作台，而不是手工编辑弹窗。
- 保存重提炼结果时，应提交完整卡片库草稿，只替换当前卡片。
- 保存重提炼结果时，应更新当前卡片的 `source.extraction_instruction`，并保留来源章节路径。
- 手工卡片编辑工作台应禁用重提炼能力。
